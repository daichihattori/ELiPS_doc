開発のルール
  1. やることをissueに書く。(すでにある場合は書かなくていい)
  2. issueに対応するブランチを作る。 例えば、issueの「#2 make a documentation」をする場合。

ブランチを作成

git branch "#2_make_a_documentation"

ブランチへ移動

git checkout "#2_make_a_documentation"

これでブランチを移動できたのであとは好きに作業してください。リモートに反映させるとき。

git add -A
git commit -m "コミット内容"
git push

このようにしてリモートブランチに反映してください。※「git push origin master」はやめてください。masterブランチにはプッシュしないでください。やっちゃっても復旧できるので必ず報告してください。

  1. 作業が終わった時は、管理者に連絡してください。mergeは管理者に任せます。
